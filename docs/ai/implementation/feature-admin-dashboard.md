# Implementation Guide — Admin Dashboard

## Development Setup
**How do we get started?**

1.  **Create App**: Use Vite to scaffold the app:
    ```bash
    cd apps
    pnpm create vite admin --template react-ts
    ```
2.  **Add to Workspace**: Ensure `apps/admin` is in `pnpm-workspace.yaml` (should be automatic with `apps/*`).
3.  **Install Dependencies**:
    ```bash
    pnpm add lucide-react react-router-dom@6 @tanstack/react-query @tanstack/react-table zod react-hook-form @hookform/resolvers clsx tailwind-merge
    pnpm add -D tailwindcss postcss autoprefixer
    ```
4.  **Initialize Tailwind**:
    ```bash
    npx tailwindcss init -p
    ```
5.  **Initialize shadcn/ui**:
    ```bash
    npx shadcn-ui@latest init
    ```

## Code Structure
**How is the code organized?**

- `src/api/`: Domain-specific API service files using a shared custom fetch client.
- `src/components/`:
    - `ui/`: Components generated by shadcn/ui.
    - `layout/`: Sidebar, Topbar, AppLayout.
    - `shared/`: Generic components like `DataTable`, `ConfirmDialog`, `FormItems`.
- `src/hooks/`: Custom hooks for data fetching and common logic.
- `src/pages/`: Page components categorized by feature (products, users, etc.).
- `src/types/`: TypeScript interfaces/types matching backend entities.

## Implementation Notes
**Key technical details to remember:**

### API Interceptor
Implement a request interceptor in `src/api/client.ts` to inject the JWT from `localStorage` into every request. Add a response interceptor to handle `401 Unauthorized` errors by clearing the token and redirecting to `/login`.

### Forms with Zod
Use Zod schemas to define validation logic. This allows for both compile-time type safety and runtime validation. Example:
```ts
const productSchema = z.object({
  name: z.string().min(1, 'Name is required'),
  price: z.number().positive(),
  // ...
});
```

### Data Table Component
Wrap `@tanstack/react-table` in a reusable `DataTable` component that accepts columns and data, and handles basic pagination/sorting UI using shadcn/ui components.

## Integration Points
**How do pieces connect?**

- **Auth**: Uses the backend `/auth/login` and `/auth/me` endpoints.
- **Catalog**: Connects to the Product and Category modules in the NestJS API.
- **Images**: Multipart form data POSTed to the backend `/storage/upload` endpoint.

## Performance Considerations
**How do we keep it fast?**

- **Query Caching**: Leverage TanStack Query's defaults for caching server state.
- **Optimistic Updates**: Implement optimistic updates for simple actions like toggling a product's publish status.
- **Tree Shaking**: Ensure Tailwind and shadcn/ui are properly configured to remove unused styles/components.

## Security Notes
**What security measures are in place?**

- **JWT Expiry**: Handle token expiration gracefully on the client side.
- **Role Guards**: Use a `ProtectedRoute` component to visually block access to admin routes for non-admin users.
- **Input Sanitization**: Rely on Zod and Backend validation to prevent XSS/SQL Injection.
